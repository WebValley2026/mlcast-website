Model Architectures

neurology

ConvGRU

Built-in encoder-decoder with 5 ConvGRU blocks. Encoder uses PixelUnshuffle for downsampling; decoder unrolls in latent space and materializes frames via PixelShuffle. Supports ensemble prediction with configurable ensemble size.

extension

Custom Networks

Plug any external architecture by replacing cfg.pl_module.network with a fdl.Config node. Your model only needs a forward(x, steps, ensemble_size) method. Example integration with mfai.HalfUNet available.

terminal

CLI & Configuration

Single Command

All training functionality is accessible via the mlcast train CLI. Configuration uses Google Fiddle for composable, reproducible experiments.

# Train with defaults
mlcast train

# Override parameters inline
mlcast train --config set:data.batch_size=32 --config set:trainer.max_epochs=50

# Use a semantic fiddler
mlcast train --config fiddler:use_random_sampler --config fiddler:use_mlflow_logger

# Reload a saved experiment
mlcast train --config path/to/saved/config.yaml
Fiddlers
  • check use_random_sampler — on-the-fly random sampling
  • check use_mlflow_logger — MLflow tracking
  • check use_anon_s3_dataset — anonymous S3 access
  • check toggle_masking — mask-aware loss
  • check set_variables — input variable selection
Config Consistency
  • sync Input channels match dataset variables
  • sync Width divisible by 2num_blocks
  • sync Ensemble models use CRPS/AFCRPS loss
  • sync Mask flag synchronized with loss
Experiment Tracking
  • assessment TensorBoard (default)
  • assessment MLflow (optional)
  • assessment W&B (optional)
  • assessment YAML config persistence + reload
speed

Training Pipeline

bolt Lightning-Powered

The training loop is built on PyTorch Lightning with ModelCheckpoint, EarlyStopping (patience=100), and LearningRateMonitor. Defaults: 100 epochs, batch size 16, Adam lr=1e-4 with ReduceLROnPlateau scheduler.

grid_view Data Pipeline

Zarr-backed PyTorch datasets with CF-convention variable resolution. Supports precomputed CSV sampling and on-the-fly random sampling. Features 70/15/15 temporal splits, random augmentation (rotations + flips), and Marshall-Palmer Z-R normalization.

Quick Start: Train Your First Model

1. Install

Create a virtual environment and install MLCast with the datasets package.

$ uv venv && source .venv/bin/activate
$ uv pip install mlcast mlcast-datasets

2. Prepare data

Sample a training-ready Zarr store from an Intake catalog source.

$ mlcast-dataset-sampler \
  --source dmi_dk_radar_precipitation \
  --output ./data/radar.zarr --time-range 2020-2024

3. Train

Launch a run, overriding parameters inline. See CLI & Configuration above for fiddlers and full options.

$ mlcast train --config fiddler:use_random_sampler \
  --config set:trainer.max_epochs=50
lightbulb

Reload any saved experiment with mlcast train --config path/to/config.yaml for full reproducibility.
